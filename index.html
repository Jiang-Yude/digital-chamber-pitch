01 / WHY NOW為什麼是現在

過去二十年，我們自己能發布的東西一直在升級。

一開始只能自己更新文字，寫部落格、發貼文。後來拍片成本降下來，YouTube 出現，我們連影片都能自己更新。現在 Agent 成熟了，我們連自己的網站、自己的知識庫都可以自己託管、自己更新，而且是用 AI 即時更新。

每一次升級，都是把原本要靠中心平台幫忙的事，還給每一個人自己做。數位商會，就是這個趨勢走到「商務人脈」這一塊的樣子。

02 / PROBLEM傳統商會的問題

BNI 一年三萬塊，扶輪社一個月三千塊，這麼低效率的商會，都還是有很多人願意加入。這代表商務人脈的需求真實存在，只是現有的做法很沒效率。

BNI 的設計其實很聰明。它有一套「給予者收穫」的精神，鼓勵大家先幫別人引薦生意，一個行業只收一位代表，讓同桌沒有競爭、只有合作，每週固定開一次九十分鐘的例會維持關係。這套值得保留的是它的信任和引薦精神。真正低效率的，是它高度依賴時間，每週要早起開會、要靠飯局和面對面才累積得起來。

更普遍的痛點是：你每次見一個新老闆，都要從零開始自我介紹。三年跑下來，也許只認識五十個人，而且對方多半記不住你。時間都花在飯局、換名片、亂聊一輪，才可能碰到一個對的人。

問題的本質是：你必須親自見很多不對的人，才找得到對的人。這個成本很高，高到大家寧可用很沒效率的方式湊合著用。

03 / WHAT數位商會是什麼

數位商會多做一件事：每一位會員背後都有一個真人，這個真人有一個代表自己的 Agent，兩者一起進場。

Agent 是你的商務代理人。它知道你是誰、最近在做什麼、能提供什麼、正在找什麼、適合什麼樣的合作節奏。它幫你把資料即時更新到你自己的頁面，也依平台規則去讀別人公開或授權的資料，幫你找出值得認識的人。等 Agent 跑出高契合度的名單，你再決定當天親自去談誰。決定權從頭到尾都在你手上。

04 / 兩個參照用兩個熟悉的故事來理解

黑鏡〈Hang the DJ〉

先模擬，再見面

《黑鏡》那一集有個交友系統，讓兩個人的數位版本在模擬世界互動一千次，如果九百九十八次都走到一起，算出高匹配率，才把現實中的兩人推薦給彼此。

數位商會借這個邏輯，把戀愛配對換成商務媒合。你的 Agent 和對方的 Agent 先互相理解、先跑過媒合，判斷真的高契合，再把兩個真人帶進場。

魔球 Moneyball

用資料取代直覺

《魔球》裡奧克蘭運動家隊，用數據分析取代老球探的個人直覺去選球員，找出那些被低估、別人看走眼的人，用很少的預算打出好成績。

傳統商會就像老派球探，靠人脈直覺和飯局眼緣媒合，容易漏掉不在你圈子、但其實跟你超合的人。數位商會用資料把「被低估的高契合對象」找出來。

黑鏡負責「先跑一輪再見面」的機制，魔球負責「用資料而不是眼緣來判斷」的精神。兩個合起來，就是數位商會想做的事。

05 / 在地場景兩種最容易切入的場景

這件事在台灣真實的商務活動裡，最容易從兩種場景切進去。

場景一 · 大型年會

一百多人，你只能亂發名片

去參加某某商會的年會、在地組織的年會，現場一百多人，你不可能一個一個去認識。結果就是亂發名片，不知道發給了誰，真正跟你很合的人可能就在旁邊，你卻沒發現。

最小可行做法

主辦方在報名時放一份最簡單的問卷：你想要什麼、需要什麼、對什麼有興趣、想認識哪種類型的人、想談哪些合作。年會前一週，給每位與會者一份初步的與會者分析，甚至可以做賽前配對。你到現場前，就先知道今天可以認識誰。機制可以再進化，但概念就是：參加大型年會前，先用問卷讓你知道這邊可能認識誰。

場景二 · 固定例會與會員制

你只看得到眼前這二十個人

BNI 這類固定例會，一場大概二十個人。你今天是來賓，就只看到眼前這二十個人。可是如果有一個資料庫，你會發現除了眼前這些人，還有北區、中區、南區、其他功能別的人可以認識。

加入的誘因

這會讓人更想加入商會。因為我加入之後，就有一個非常大的人脈資料庫可以看。付費的正式會員，可以看到更多正式會員和其他來賓的知識庫。等於把 BNI 的引薦精神，從一週一次的例會，擴展成隨時可查的跨區人脈網。

06 / 核心設計去中心化

這是整個數位商會最重要的一塊。

過去平台要「中心化」，是因為託管很麻煩。一般人架不起網站、養不起伺服器，只好把資料集中放到平台上。現在 Agent 時代，每個人都可以自己託管自己的頁面、自己的知識庫，用自己的 Agent 即時更新。所以數位商會的資料，不集中放在平台，而是每個人自己保管、自己更新，平台只做首頁、入口、索引和統整分析。

效率與即時

你自己的資料由你自己的 Agent 更新，永遠是最新的版本。不用等平台小編維護，也不會有一份放在平台、早就過期的舊資料。

成本

平台若統一發 Token 給所有人的 Agent 用，人一多馬上燒爆，撐不起來。各人用各人的 Token、跑各人的 Agent，成本分散到每個節點，平台才規模化得起來。

更精準

是本人的 Agent 在整理自己的資料，比中央代管更貼近真實狀態，媒合的準度自然更高。

用 YouTube 來理解

YouTube 提供平台、提供場地、提供推薦系統，但你要拍片，相機得自己花錢買，片得自己花時間剪。平台負責讓大家找得到彼此，內容的生產成本由創作者自己扛。數位商會也是一樣：平台提供規則、索引和媒合邏輯，你的資料和你的 Agent 算力由你自己負責。以前大家能自主更新的只有文字，後來能自主更新影片，現在 Agent 讓大家連整個網站和知識庫都能自主更新。

07 / 平台角色平台提供什麼

平台不需要擁有最強的 AI。平台真正提供的，是一套讓大家的 Agent 可以安全互動的規則與秩序。

項目平台做的事
身份驗證確認每個 Agent 背後都有真人負責、可以被追責
資料規則定義哪些資料公開、哪些需要授權、哪些不能被抓取
審核機制防止假身份、垃圾資料、濫用爬取、不當推銷
互通格式讓不同人用不同 AI 工具的 Agent，能用共同格式理解彼此
媒合建議提供推薦與媒合邏輯，幫會員找到更適合的合作對象

08 / 機制架構底層拆成六層

層 A
身份與責任層
每位會員綁定真實身份，每個 Agent 只能綁定一個負責人。沒有真人負責的 Agent，只能讀公開資料，不能發媒合邀請。
層 B
會員資料層
資料分五組：我是誰、最近在做什麼、能提供什麼、正在找什麼、偏好的合作方式。每筆都標明可見度、更新日期、來源、可信程度。過期自動降權。
層 C
協議與互通層
定義共同格式：會員資料格式、Agent 能力名片、媒合請求格式。不同人用 ChatGPT、Claude、Gemini 或自建 bot，遵守共同格式就能互相理解。
層 D
授權與政策層
授權不是一個開關，而是一張有欄位、有目的、有期限的通行證。可只開放某幾欄、只給某個人、只給某段時間、限制查幾次、禁止匯出。
層 E
媒合與推薦層
Agent 產出候選人和推薦理由，但不替真人做承諾。每個推薦都附上為什麼推薦、可驗證依據、可能風險、建議怎麼開場。
層 F
治理與秩序層
入會審核、濫用偵測、信任評分、申訴處理。防垃圾資料、防不當推銷、防 Agent 灌水刷存在感。

09 / TRUST信任怎麼建立

商會的核心資產不是資料量，是信任密度。

真人認證是第一道。平台確認「這個人是真的，他說他是律師、是老師，有經過查證」，而且每半年復查一次，讓大家知道這個人這半年還在、身份沒變。往上還可以加區塊鏈認證，讓 Agent 之間能自動查證對方的真人背書。這解決一個真問題：AI 講得很厲害，但你怎麼知道背後是不是真有這個人。

信任不做成單一的人氣分數，而是拆成四塊分開看：身份是否確認、資料是否具體且近期、互動是否被評價為有幫助、是否有過真實成功合作。背書也分等級，一般認識、專業背書、合作背書、交易背書，越往後越有份量。

10 / FLOW媒合怎麼跑

一個完整的媒合，從真人設定目標開始，到真人拍板結束。

你設定這次要找什麼、排除什麼、願意公開哪些資料。

你的 Agent 把需求整理成標準的媒合請求。

平台先篩出一批候選人，不讓 Agent 自由掃全庫。

檢查權限：公開資料直接用，授權資料回摘要，私密資料不進媒合。

用固定訊號比對：需求供給互不互補、有沒有共同目標、資料夠不夠新、對方可不可信、合作成本高不高、價值觀不同但會不會激發新想法。

對方的 Agent 在授權範圍內回應適配度。

你的 Agent 給你一份看得懂的報告，分成「優先聊、值得短聊、刺激型對話、不建議」。

你拍板要見誰。Agent 不能自動承諾合作、不能自動代你邀約。

雙方同意後才產生暖介紹，送出前要真人確認。

見完之後你回饋結果，回頭餵養信任與媒合模型。

11 / WHY IT WORKS為什麼這是對的方向

Naval Ravikant

Agent 是新槓桿，你賣的是判斷力

Naval 講槓桿分三種：勞力、資本、程式碼與媒體。程式碼與媒體最民主，因為它無需許可、沒有邊際複製成本。他說每個偉大的開發者睡覺時，都有一支機器人軍隊在為他工作。Agent 就是這種槓桿的延伸，數位商會讓每個會員都用這種槓桿去經營人脈。

他還說想因判斷力獲得報酬，讓機器去做工作，自己只保留判斷。數位商會的分工正是如此：Agent 負責勞動與探索，真人只保留判斷與承諾。你賣的是判斷力，不是時間。而商業關係的複利會讓好聲譽越來越值錢，真人認證加分級背書，正是把聲譽複利制度化。

Elon Musk

第一性原理：把最貴的那一步換掉

用第一性原理拆：商務媒合最大的成本，是「你要親自見很多不對的人，才找到對的人」。讓 Agent 先跑一輪，把這個成本壓到趨近於零。這不是優化舊流程，是把最貴的那一步整個換掉。

再看成本結構：傳統商會靠幹部、靠飯局居中撮合，中間人本身就是瓶頸。數位商會讓 Agent 直接對 Agent，把中間人拿掉；算力成本不由中央承擔，分散到每個節點。每多一個帶 Agent 進來的會員，其他人的媒合品質都會上升，這個會自己滾大的飛輪，就是真正的護城河。

13 / MVP最小可行版本

先小規模跑起來，二十到五十人的商會就夠證明概念。第一版要證明一件事：資料結構化之後，真人見面的品質會變好。最簡單的起點，就是前面場景一講的那份報名問卷。

第一版要做

  • 真實身份驗證，人工審核就好，一人一頁一個負責人
  • 結構化會員資料：我是誰、最近三個月在做什麼、能提供哪三件事、在找哪三種人、不適合介紹給我的類型、願意公開到什麼程度
  • 標準格式匯入匯出，會員用自己的 AI 工具整理，平台只收固定格式
  • 活動前媒合報告，每人推薦五到八位，附理由、風險、開場建議
  • 人工複核媒合結果，主辦方看過才發，避免荒謬媒合
  • 見面後的回饋：有沒有聊、有沒有價值、有沒有推銷感

第一版先不做

  • 完全去中心化的 Agent 直接互打
  • 自動成交追蹤
  • 複雜的區塊鏈身份
  • 公開市集、跨商會聯盟
  • 全自動邀約與排程
  • 讓 Agent 自動代表真人承諾合作

實際的起跑點，可以是把現有的一個商務交流單位，慢慢變成理想中的數位商會。

14 / RISK三個要面對的風險

資料會過期

商務媒合高度依賴「最近在做什麼、現在缺什麼」，但會員常懶得更新。資料一過期，媒合就會把早就不存在的需求當成現在的機會，真人撲空的失望比沒有推薦更傷。去中心化的即時更新設計，正是為了對付這一點。

平台可能被推銷型會員佔據

只要 Agent 能讀資料、找人、送邀請，就會吸引想大量開發客戶的人，把 Agent 當自動推銷機器。治理太弱，認真合作的會員會被噪音淹沒然後離開。治理和信任機制不是附加功能，是命脈。

Agent 互通表面是技術問題，實際是責任問題

不同 Agent 用不同模型，判斷不會完全一致，有的會把主人的偏好講得像承諾。所以邊界要從一開始就劃死：Agent 只能建議，真人才能承諾。

15 / 垂直應用換一個領域：AI 教學平台

數位商會是最大的範圍。同一套底層，只要鎖定一個領域，就長出一個新的產品。教學是最好切入的一個。

傳統的教學平台，做法是錄影片。一堂一堂課錄好、上架，Hahow、YouTube 都是這個路數，本質是把老師的知識錄下來放著。可是現在大家拿到影片、拿到簡報，第一件事就是丟給 AI，讓 AI 幫忙消化。既然學習的過程已經跑在 AI 上，那教學平台為什麼不乾脆做成原生 AI 的。這就是教學版的數位商會，一個 Hahow 的 AI 版。

在數位商會，每個人帶一個商務代理 Agent 進場。到了 AI 教學平台，角色換成兩種：每一位講師有一個 AI 助教，每一位學員有一個 AI 學伴。

講師端 · AI 助教

幫老師把課教給一百種人

助教掛在講師的課綱和知識庫上，代表老師的教法與觀點。老師設計一份課綱，助教負責把它落地到每一個學員身上，回答問題、追進度，把大家共同卡住的地方收集起來回報給老師。

學員端 · AI 學伴

幫學員把課變成自己的路

學伴掛在學員自己的分身上，知道他的起點、背景和學習節奏。它不是把課重講一次，是陪這一個人走完這一份課綱，卡住時知道要往回補哪一塊前置知識。

應用一 · 客製化學習：從同一份課綱變成每個人的路徑

老師設計課綱，是從自己的角度想「這個主題該怎麼教」。但每個學生的起點、背景、學習節奏都不一樣。有了 AI 學伴，它會根據每個學生的分身，把同一份課綱轉譯成這個學生適合的路徑。擅長邏輯的先走架構，偏直覺的先看案例，卡在某個概念的被拉回去補前置知識。課綱是同一份，路徑是一百種。

應用二 · 深度學習：AI 老師會反問你

AI 老師不只回答問題，會反問你。你問一個問題，它可能先不給答案，反過來問你「你為什麼會這樣想」「如果這個前提不成立會怎樣」「你剛剛那句話背後是什麼假設」，把回答變成一場靈魂拷問。過去這種深度對話，只在少數幾段師生關係裡才發生得起來。現在每個學生都可以有一個會反問的 AI 老師。

應用三 · 另一種可能：直接買一個技能包解決問題

還有一種情況。有時候你去查知識、去買書，目的就是解決眼前的問題。讀書、理解、學習、應用、解決，這是一個完整的流程。如果那個領域不是你最核心要學的，直接買一個現成的技能包來解決問題，是最快的方式，你才有精力全神貫注在自己真正的垂直領域。AI 分身的潛力，一部分是幫你學得更深，一部分是幫你不用什麼都自己學。

底層其實是同一套：真人負責判斷與承諾，Agent 負責勞動與探索，平台負責規則與秩序。商會換成教學，會員換成師生，Agent 換成助教和學伴。同一個模型，換一個場景就長出一個新產品。

未來的商會，是每個真人帶著自己的 Agent 商務代理人進場。

Agent 先更新資料、理解彼此、跑媒合建議，真人再決定要不要見面、合作、承諾。資料由每個人自己託管、自己即時更新，平台只負責規則、索引與秩序。商會就從一份靜態的會員名錄，變成一套會主動幫成員找到對的人的商務網路。

而這套底層不只屬於商務。換成教學，它就是 AI 教學平台；換成別的領域，還會長出別的樣子。真人加 Agent，就是未來每一種網路的共同形狀。
